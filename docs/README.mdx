## 01. Phase 1 – Core Launch (Bay Area, Residential-Focused – Mixed Persona in Ads)

Phase 1 = **Bay Area only**, no explicit persona split in campaigns (Residential + Commercial mixed), but the form is already “future-proofed”.

<details>
<summary><strong>01.1 – Segmentation & Taxonomy (Global)</strong></summary>

These are **global** docs, but heavily used when building Phase 1.

- **02_01_segmentation-taxonomy.mdx**  
  - Canonical definitions for:
    - Surfaces (Deck, Siding, Wood Structures, Furniture, Pressure, Catch-All).
    - Geos (Bay Area vs Fringe) – used from Phase 2 onward.
    - Personas (Residential vs Commercial) – used from Phase 3 onward.
  - Naming conventions used across CSVs and LP specs.

- **02_02_full_canonical.mdx**  
  - Full canonical tables for:
    - Campaign naming
    - Geo codes
    - Persona codes
    - Surface codes
  - Separate tables per phase (Phase 1 → 2 → 3), including execution-level columns (e.g., CTA, LP target, etc.).
</details>

<details>
<summary><strong>01.2 – Campaign & Keyword Payloads</strong></summary>

> These are the **execution payloads** Phase 1 is built from.  
> Paths/names may differ; update links as needed to match your repo.

- **FINAL_cal_preserving_consolidated_keywords - FINAL_cal_preserving_consolidated_keywords.csv**  
  - Master keyword set used as the base for Phase 1 campaigns.
  - Columns typically include: `Campaign`, `Ad Group Name`, `Keyword`, `Match Type`, `Ad Group Negatives`, `Campaign Negatives`.

- **phase_1_campaign-master.csv** (or `doc2/campaign_master_phase1.csv`)  
  - Canonical Phase 1 campaign mapping with columns like:
    - `campaign`
    - `surface`
    - `phase`
    - `ad_group`
    - `keyword`
    - `match_type`
    - `lp_url`
    - `cta`
</details>

<details>
<summary><strong>01.3 – Estimate Request Form (Shared Across All Phases)</strong></summary>

- **estimate_request_form_v1_FINAL.mdx**  
  - Canonical **web lead form** spec for all phases.
  - Sections:
    - Job Site (Street, City, State, Zip)
    - Property Type (Residential / Commercial)
    - Surfaces to be Addressed (multi-select: Deck, Siding, Fence/Gate, Structures, Furniture, Hardscape, Other)
    - Project Timing
    - Additional Project Description
    - Contact Info
    - Relationship to Property
    - How Did You Hear About Us
    - Other Comments
    - SMS Consent
  - Includes SFDC-friendly field names and guidance on how the form stays phase-stable (Phases 1–3).

> Even though Phase 1 ads are mixed persona, the form already collects `property_type__c` for future persona split.
</details>

## 03. Phase 3 – Persona Split (Residential vs Commercial)

Phase 3 introduces explicit **Residential vs Commercial personas** in campaigns and LPs, while still using the same global form.

<details>
<summary><strong>03.1 – Phase 3 Campaign Master & LP Mapping</strong></summary>

- **campaign_master_phase3.csv**  
  - Canonical Phase 3 campaign mapping including:
    - `campaign`
    - `persona` (`residential` / `commercial`)
    - `surface`
    - `geo` (if present in this file)
    - `lp_url`
    - `cta`
    - `sitelinks`
  - Used by both Marketing and Web to keep campaigns and LPs aligned.

- **lp_mapping_phase3.csv**  
  - Mapping from:
    - `campaign` → `persona` → `surface` → `lp_url`
  - Example pattern (no geo in URL at this stage):
    - `Deck,residential,deck,https://calpreserving.com/residential/deck/`
    - `Deck,commercial,deck,https://calpreserving.com/commercial/deck/`
    - `Catch-All,residential,catch-all,https://calpreserving.com/residential/`
    - `Catch-All,commercial,catch-all,https://calpreserving.com/commercial/`
</details>

<details>
<summary><strong>03.2 – Residential LP Specs (Per Surface)</strong></summary>

## How to Use This TOC

- **Marketing Team**  
  - Start with: `01_01_executive-context.mdx`, `02_01_segmentation-taxonomy.mdx`, `02_02_full_canonical.mdx`.  
  - Then move into `/phase_1` and `/phase_2` campaign masters and LP mappings.

- **Web Team**  
  - Start with: LP specs (`lp-spec-*.mdx`) and the form spec (`estimate_request_form_v1_FINAL.mdx`).  
  - Then align routes/URLs using `lp_mapping_phase3.csv`.

- **Salesforce Team**  
  - Start with: `estimate_request_form_v1_FINAL.mdx` and `03_01_sfdc-mapping_COMPLETE.md`.  
  - Then plug into reporting via `03_02_reporting-spec.*`.
